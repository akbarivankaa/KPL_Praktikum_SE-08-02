## Deskripsi


Untuk Bagian Konsep Dasarnya yaitu Singleton, Pola Singleton bertujuan untuk memastikan bahwa sebuah sistem hanya membuat satu instance tunggal dari suatu objek dan menyediakan satu titik akses global terhadap instance tersebut ke seluruh bagian aplikasi.

Penerapan di Kode Dalam ekosistem Node.js, ketika sebuah file  diimpor untuk pertama kalinya, kode di dalamnya dieksekusi dan hasilnya disimpan di dalam memori. Pada kode di atas, koneksi client dan instansiasi ORM drizzle dibuat, lalu diekspor sebagai konstanta db. Karena sifat caching ini, objek db secara otomatis bertindak sebagai Singleton.

disini keuntungan sistem file lain di berbagai bagian proyek, seperti di dalam backend/src/services/task.service.ts, mengimpor dan memanggil objek db yang sama persis setiap kali mengeksekusi operasi seperti getTasks atau createTask. Jika koneksi database tidak dibuat menggunakan pola Singleton ini, aplikasi akan terus-menerus membuka koneksi baru setiap kali ada request masuk dari client. Hal ini dapat menguras memori server secara drastis, memperlambat response time, dan berisiko menyebabkan file lock / database crash pada file local.db.
