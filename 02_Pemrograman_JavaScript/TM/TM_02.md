## Deskripsi

update pada kode ini, aku ubah beberapa penyesuaian di file testing dan file tm.js biar bisa pas sama yg diminta soal. Di bagian file testing, aku ubah dan samain format outputnya. Sebelumnya kan ada yang pakai koma, nah sekarang  kuubah jadi pake spasi semua biar konsisten. abis itu, aku juga rapihin bagian error-nya pakai error.message, jadi kalau ada tes yang gagal, terminal cuma nampilin pesan intinya aja dan nggak kepanjangan.

Terus buat file tm.js-nya, di baris paling atas aku tambahin validasi buat ngecek input. Jadi kalau data yang dimasukin ternyata bukan array (misalnya cuma input angka biasa atau teks string), program bakal langsung berhenti dan ngeluarin tulisan "Input tidak valid".

Lalu buat logika utamanya, aku pake perulangan buat ngecek setiap angka di dalam array. Nah, bagian yang paling penting di sini tuh urutan pengecekannya. Aku sengaja ngecek yang kelipatan 14 duluan sebelum ngecek kelipatan 7 dan 2. Soalnya, kalau kelipatan 2 atau 7 dicek duluan, angka kelipatan 14 bakal keburu masuk ke kondisi tersebut dan malah gagal nyetak tulisan "FizzBuzz".

Terakhir, buat ngerangkai hasilnya jadi satu kalimat panjang, aku bikin variabel penampung (hasil). Logikanya, setiap ada kata atau angka baru yang mau dimasukin, aku tambahin spasi di antaranya. Cara ini aku pakai biar kalimat akhirnya tersusun rapi dan nggak ada sisa spasi berlebih di awal kalimatnya
